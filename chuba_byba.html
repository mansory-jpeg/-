Особенности информационного, программного и технического обеспечения различных видов ИС

Давайте разберем особенности обеспечения для основных типов информационных систем (ИС).

Сначала вспомним устройство и функционирование ИС: это всегда комплекс взаимосвязанных компонентов (обеспечивающих подсистем), работающих вместе с людьми для сбора, обработки, хранения и распространения информации с целью поддержки бизнес-процессов, анализа и принятия решений.

Классификация ИС по уровню управления и охвату задач

Мы рассмотрим особенности обеспечения на примере четырех основных типов ИС:

  1. Системы операционного (transaction processing systems) и автоматизированные системы управления (АСУ)
  2. Информационные системы для поддержки принятия решений (Decision Support Systems, DSS) и офисы экспертов
  3. Корпоративные информационные системы (Enterprise Resource Planning, ERP)
  4. Геоинформационные системы (ГИС)

1. Системы операционного уровня (TPS) и АСУ

Назначение: Регистрация, обработка и хранение ежедневных операций (транзакций). Автоматизация рутинных задач.

Примеры: Система учета продаж в магазине, система бронирования билетов, банковские операции по счетам, АСУ "Теплоснабжение".

Особенности обеспечения:

Обеспечение Особенности
Информационное (ИО)
  • Структурированность: Данные строго структурированы, регламентированы (справочники товаров, клиентов, типов операций).
  • Большие объемы: Высокие требования к хранению большого количества записей о транзакциях
  • Актуальность: Требуется постоянное и быстрое обновление данных (остатки на складе должны меняться сразу после продажи).
  • Историчность: Данные редко удаляются, накапливается подробная история всех операций
Программное (ПО)
  • СУБД: Используются реляционные СУБД (MySQL, PostgreSQL, MS SQL Server, Oracle) для обеспечения целостности и согласованности данных (транзакции ACID).
  • Прикладное ПО: Часто разрабатывается под конкретную задачу (заказная разработка) или настраивается под нужды предприятия (на базе платформ типа 1С).
  • Интерфейс: Преобладают формные интерфейсы для быстрого ввода данных
Техническое (ТО)
  • Надежность: Требуется высоконадежное оборудование с дублированием критических компонентов (дисковые массивы RAID, резервные источники питания).
  • Производительность: Высокие требования к скорости обработки транзакций (Input/Output Operations Per Second - IOPS) и отказоустойчивости дисковой подсистемы.
  • Доступность: Система должна быть доступна в режиме, близком к 24/7.

2. Системы поддержки принятия решений (DSS) и офисы экспертов

Назначение: Анализ данных, моделирование ситуаций, подготовка отчетов для менеджеров среднего и высшего звена.

Примеры: Система анализа продаж по регионам, система прогнозирования спроса, BI-системы (Power BI, Tableau).

Особенности обеспечения:

Обеспечение Особенности
Информационное (ИО)
  • Интеграция: Данные поступают из различных источников: операционных систем (TPS), внешних баз данных, Excel-файлов.
  • Агрегированность: Данные часто предварительно агрегируются и очищаются (хранятся в хранилищах данных - Data Warehouse).
  • Многомерность: Данные организованы в виде OLAP-кубов для быстрого и гибкого анализа по множеству измерений (время, регион, товарная категория).
  • Релевантность: Важна не свежесть каждой транзакции, а корректность и полнота данных для анализа
